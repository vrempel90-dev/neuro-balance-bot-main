# Neuro Balance Python Bot — GPT-first 1:1 logic

Версия, где GPT-4o-mini ведёт живой диалог и максимально повторяет логику старого бота Neuro Balance.

## Архитектура

- GPT-4o-mini — понимает клиента, язык, жалобу, намерение, отвечает как живой администратор.
- Python — только контроль и исполнение: Wazzup, режим 20:00–08:00, CRM, запись, перенос, отмена, оператор, outcome, безопасность.
- CRM API — patient-lookup, doctors, services, check-slots, book, cancel, reschedule, escalate, outcome.

## Важно

- Бот днём молчит и не мешает КЦ.
- На профильные жалобы сначала отвечает по смыслу, потом ведёт к записи.
- На непрофильные жалобы не записывает автоматически, передаёт оператору.
- Не выдумывает врачей/слоты/записи — только через CRM tools.

### v28 Strict TZ Stability
Стабилизация по реальным ночным диалогам и ТЗ:

1. Язык
- Если клиент пишет по-русски, бот не должен отвечать на казахском.
- Русские вопросы про Астану, адрес, цену, график, процедуры всегда остаются русскими.
- Исправлен залипший язык из предыдущего состояния диалога.

2. Обычная заявка на запись
- На `хочу записаться`, `хочу записаться по акции`, `хочу на диагностику` бот не пишет `передам координатору`.
- Бот не пишет пациенту `у вас нет активной записи` на новую заявку.
- Правильный старт: приветствие + вопрос `что Вас беспокоит?`.

3. Цена/стоимость
- Вопросы `сколько стоит`, `стоимость лечения`, `сколько стоит лечебная процедура` обрабатываются отдельно.
- Сначала ответ на вопрос, потом возврат к текущему шагу записи.

4. Передвижение / 68 лет / сложные случаи
- 68 лет не является причиной отказа.
- Если пациент пишет, что тяжело ходить — бот уточняет, сможет ли он самостоятельно прийти.
- Если пациент отвечает `да` — бот продолжает запись, а не отказывает.
- Если пациент отвечает `нет / не смогу` — мягко передаёт администратору, без грубого `мы не сможем взять пациента`.

### v29 Real Night Dialog Fixes
Правки по реальным ночным скринам:

1. Жалобы не превращаются в отказ
- `грыжа`, `протрузия`, `нога после перелома`, `бел омыртқа жарығы болуы мүмкін` — это жалоба, а не противопоказание.
- Бот не должен писать `не могу записать вас на консультацию`.
- Правильный ответ: можно прийти на первичную консультацию, врач осмотрит, далее вопрос про возраст.

2. `Ничего` не зацикливает диалог
- Первый ответ `ничего` — мягкое уточнение зоны проверки.
- Второй ответ `ничего` — бот ведёт как профилактическую консультацию и спрашивает возраст.
- Больше нет бесконечного повторения `что Вас беспокоит?`.

3. Вопросы после списка слотов
- Если слоты уже были показаны, бот не повторяет весь список снова.
- На вопросы `какое лечение`, `сколько стоит`, `адрес`, `график` сначала отвечает по сути.
- Потом возвращает к шагу: `Какое время из вариантов выше Вам удобно?`.

4. Отказ от записи
- `керек емес`, `не надо`, `не нужно`, `передумал` останавливают запись.
- Бот не продолжает давить вопросом `какой день удобен`.

5. Цена 5000
- `консультацию 5000 тг ма?`, `5 000`, `тг`, `тенге` распознаются как вопрос о цене.
- Бот отвечает про первичный приём 5 000 тг и возвращает к текущему шагу.
